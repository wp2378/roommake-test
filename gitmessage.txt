Feat: Update Settings
# 형식 - [type]: [title]
#
# [type] 입력 목록
#  1. Feat: 어떤 특정 어플리케이션에 더할 새로운 feature
#  2. Fix: 어떤 오류 해결 (fix)
#  3. Style: 스타일과 연관된 feature나 업데이트들
#  4. Refactor: 코드 베이스의 특정 부분을 재정렬 (refactoring)
#  5. Test: 테스트와 관련된 모든 것
#  6. Docs: 문서화에 관한 모든 것
#  7. Chore: 정규 코드 유지보수
#
# [title] 규칙
#  1. 50자 이내 짧은 요약 (영어의 경우 첫글자는 대문자로)
#  2. 명령형으로 작성 (과거형X)
#  3. 제목 끝에 .(마침표) 금지


# [Body] 작성 규칙
#  1. 제목과 본문 사이 빈 칸 한줄 필수 (제목과 본문 분리)
#  2. 생략 가능, 필요 시 자세한 설명 작성 (72자 내외로 마무리)
#  3. 적용한 변경사항과 그것을 만든 이유에 대해 설명
#     - 어떻게보다는 "무엇을", "왜"
